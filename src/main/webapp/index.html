Starter Code Backend (How to)

(Deploy the backend before the frontend)

Follow the guide in the repository README.md.
File overview
src.cors
  • Handles CORS-filters
src.enitites
  • Handles project entities (if any)
    • Includes Role, User for testing hashing
src.errorhandling
  • Handles errors through DTOs and mappers
src.facades
  • Contains facades for user and API handling
src.rest
  • Handles API endpoints, included are:
    • User (dependant on role) login
    • The combined 5 endpoints from SWAPI
src.security
  • Handles JWT, SecurityContext and RolesAlloved for certain endpoints
src.utils
  • Handles database and user setup*

*It is important to run this script once and not to push it, read the readme for more info.

src.main/resources/config.properties
  • .properties file that handles local/remote database setup

Default endpoints

  • Server status: api/info/
  • Count all users: api/info/all

Endpoints continued
(requires login - use frontend)
  • Star Wars API: Five endpoints in one: api/info/five
  • User role endpoint: api/info/user
  • Admin role endpoint: api/info/admin
  • Admin & User role endpoint (either or): api/info/admin
  • SuperUser endpoint (experimental, deprecated): api/info/bothrestricted
